getElementsByTagName和querySelectorAll

getElementsByTagName和querySelectorAll我们在DOM中经常用到，今天我们来分析下它的细节。

A.案例

// HTML
<h1 id="parent">
    <div>1</div>
    <div>2</div>
</h1>

// JS
<script type="text/javascript">
    var parent = document.getElementById('parent');
    var child_nodes = parent.children;
    console.log(child_nodes.length);
    parent.appendChild(document.createElement('div'));
    console.log(child_nodes.length);
</script>

断点调试下，我们可以发现，当给parent添加一个div节点的时候，数组中div的数量就发生了变化！

B.解析

element.getElementsByTagName

The Element.getElementsByTagName() method returns a live HTMLCollection of elements with the given tag name. The subtree underneath the specified element is searched, excluding the element itself. The returned list is live, meaning that it updates itself with the DOM tree automatically. Consequently, there is no need to call several times Element.getElementsByTagName() with the same element and arguments.

从API文档解释来看，getElementsByTagName根据标签名获取的是实时的HTMLCollection集合。当DOM树更新时，HTMLCollection集合也会实时更新。

element.querySelectorAll

Returns a non-live NodeList of all elements descended from the element on which it is invoked that matches the specified group of CSS selectors. (The base element itself is not included, even if it matches.)

而querySelectorAll返回的不是实时的NodeList结合。

HTMLCollection和NodeList

An HTMLCollection in the HTML DOM is live; it is automatically updated when the underlying document is changed.

从API上来看，HTMLCollection 集合是实时的。而NodeList大多情况也是实时的，只有少数情况（document.querySelectorAll）返回一个静态的NodeList

In some cases, the NodeList is a live collection, which means that changes in the DOM are reflected in the collection. For example, Node.childNodes is live:
In other cases, the NodeList is a static collection, meaning any subsequent change in the DOM does not affect the content of the collection. document.querySelectorAll returns a static NodeList.
